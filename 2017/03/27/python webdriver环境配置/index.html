配置python webdriver运行环境

WebDriver是主流Web应用自动化测试框架，具有清晰面向对象 API，能以最佳的方式与浏览器进行交互。

Selenium WebDriver 又称为 Selenium2。

Selenium 1 + WebDriver = Selenium 2

文章背景

最近接到了一个为其他组同事写个自动控制网站的工具，原本是要用chrome插件的方式来控制网站，研究了一周的时间，然而呢，狗血的事情发生了，结果高通的这个网站用到了java小程序-applet这个东西，他需要浏览器有java环境，NPAPI是java小程序依赖的一个东西，chrome浏览器已经从2015年的4月发布的42版本开始就不支持java jre环境了，网上资料说到45版本之前还可以通过启用插件的方式来弥补，不过现在已经是57版本了，太老的chrome版本根本不能用了，所以只能换一种方式来实现工具的功能。知道兄弟我当时的艰辛了吧，这个工具恐怕做不出来，晚上睡觉都在想！好了，开始步入正题：

使用python
  1. 输入 python 即可使用新版本的 python；
  2. 系统自带的脚本会以 /usr/bin/python 的方式直接调用老版本的 python，因而不会对系统脚本产生影响；
  3. 使用 pip 安装第三方模块时会自动按照到当前的python版本下，不会和系统模块发生冲突。
  4. 使用 pip 安装模块后，可能需要执行 pyenv rehash 更新数据库；
